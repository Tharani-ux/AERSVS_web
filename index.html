AERSVS System

Step 1: Scenario 1.1 Automated Triggering (AT)

When a car travels fast and suddenly crashes, the MEMS system detects the crash and triggers the AERSVS system within 10 to 30 milliseconds.

Process Explanation

MEMS sensors detect the crash using accelerometers, brake sensors, and seatbelt sensors. The information is transferred via the CAN bus network.

Setup 1: Microcontroller/Processor

The microcontroller bridges the car’s CAN network and the Iridium modem, reading CAN data and sending it via UART.

Setup 2: CAN Bus Integration

  • CAN Transceiver connects to the car’s CAN bus.
  • CAN_H and CAN_L connect to the CAN bus.
  • The microcontroller communicates with the transceiver via TXD and RXD.

Setup 3: Iridium 9603 Modem Connection

  • Power supply: 3.2V–5.5V
  • UART interface for TXD and RXD communication
  • Optional CTS/RTS flow control

Iridium 9603 Data Transfer Capabilities

Designed for low-bandwidth, global data communication via the Iridium satellite network.

Max Uplink: 340 bytes per message

Max Downlink: 270 bytes per message

Latency: 20-60 sec per message

Challenges & Solutions

  • Filter important data to fit within the 340-byte limit.
  • Compress messages before transmission.
  • Use batch transmission instead of real-time.

Scenario 1.2: Manual Triggering (UKB Button)

The system can be manually triggered in case of emergencies via the Uyire Kapan Button (UKB), which is located in a safe position inside the car.

Step 2: Server Communication

Data from the Iridium modem is sent to the server and categorized based on emergency type.

Case 1: 112 Services

  • Channel 1: Voice call
  • Channel 2: SMS
  • Channel 3: Email
  • Channel 4: Web portal

Case 2: Nearest Emergency Services (NES)

The system finds the nearest emergency services based on GPS coordinates and government databases.

Step 3: Emergency Contact Notification

After confirming emergency detection, the system notifies predefined contacts via:

  • Voice call
  • SMS

Step 4: Mobile Application Features

The mobile app enables real-time monitoring and alerts.

VtoO Data Parameters

  • Vehicle Speed
  • Engine RPM
  • Fuel Level
  • Battery Voltage
  • Brake Status
  • Airbag Status
  • Diagnostic Trouble Codes (DTCs)

Total Estimated Data Size Per Second
