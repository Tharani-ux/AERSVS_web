AUTOMATED EMERGENCY RESPONSE SYSTEM VIA SATELLITE

IDEA

Step 1:

Scenario 1.1 Automated Triggering (AT)

When a car travels fast and suddenly crashes, the MEMS system detects the crash and triggers the AERSVS system within 10 to 30 milliseconds.

Scenario 1.2 Manual Triggering (UKB Button)

The system can be manually triggered in case of emergencies via the Uyire Kapan Button (UKB), which is located in a safe position inside the car.

Process Explanation

  • MEMS sensors detect the crash using accelerometers, brake sensors, and seatbelt sensors.
  • The information is transferred via the CAN bus network(digital and analog communication ports) to the AERSVS.
  • AERSVS transfer the data to satellite using Iridium satellite modem.

Iridium 9603 Data Transfer Capabilities

Designed for low-bandwidth, global data communication via the Iridium satellite network.

Step 2:

Server Communication

Data from the Iridium Ground station is sent to the server and categorized based on emergency type.

  • Case 1: 112 Services
  • Case 2: Nearest Emergency Services (NES)

Step 3:

Emergency Contact Notification

After confirming emergency detection, the system notifies predefined contacts via:

  • Voice call
  • SMS

Step 4: Mobile Application Features

The mobile app enables real-time monitoring and alerts.

  • VtoO Data Parameters (vehicle to Owner)
  • VtoS Data Parameters (vehicle to Server)
